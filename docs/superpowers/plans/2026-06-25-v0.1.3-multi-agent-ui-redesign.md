# v0.1.3 Multi-Agent UI Redesign Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the single implicit "default agent" model with a first-class multi-agent system: an `agents` table + CRUD API, a two-section accordion sidebar (Agents / Chats), full-width single-view switching (empty / chat / agent config), and live-spec message sending where each message uses the agent's *current* config.

**Architecture:** Full-stack, layered bottom-up. Shared TypeScript types change first, then the API's idempotent migration sequence gains the `agents` table + backfill, then the `PgChatStore` gains agent CRUD and agent-bound session creation, then the Express API exposes agent endpoints and switches message-send to fetch the live spec, then the web `api.ts` wrapper, then the `App.tsx` UI restructure. Each layer is independently tested with the existing harnesses (`pg-mem` for API/store, `@testing-library/react` + `vitest` for web).

**Tech Stack:** TypeScript, pnpm workspaces, Express, `pg` + `pg-mem` (tests), `nanoid`, Zod (`agentSpecSchema`), React 18, `react-markdown`, Vitest, `@testing-library/react`, `@testing-library/user-event`.

## Global Constraints

- **Test runner:** Vitest. Root: `pnpm test` runs `vitest run`. Typecheck: `pnpm -r typecheck`. API only: `pnpm --filter @agent-builder/api test`. Web only: `pnpm --filter @agent-builder/web test`.
- **pg-mem safety:** All DDL added to `runChatMigrationsSequence` MUST be idempotent and pg-mem-safe. Reuse the existing helpers (`createTableIfNeeded`, `addColumnIfNeeded`, `createIndexIfNeeded`) and wrap any unsupported op in try/catch matching `isPgMemUnsupportedIfNotExists` (`error.message.includes("Not supported")`).
- **`alter column ... set not null` ordering:** Only run `set not null` AFTER every row has a non-null value. Backfill `chat_session.agent_id` before the `set not null` step.
- **Agent name/description are derived:** `name = spec.identity.name`, `description = spec.identity.description`. Single source of truth. This holds in migration seeding, `POST /api/agents`, and `PUT /api/agents/:id`.
- **API key handling:** API keys remain runtime-only frontend state, passed per message in `runtimeSecrets.apiKey`. Never persisted. Strip `apiKey`/`apiKeyRef` from every spec returned in an HTTP response via the existing `publicAgentSpec()` helper (defined in `apps/api/src/index.ts`).
- **Snapshot policy:** Stop writing `chat_session.agent_spec_snapshot` for new sessions (pass NULL). KEEP writing `agent_tasks.agent_spec_snapshot` at task creation (captures the just-fetched live spec). Do NOT drop either column in v0.1.3.
- **`ChatSession.agentSpecSnapshot`** becomes optional: `AgentSpec | null`.
- **`Agent` type shape:** `{ id: string; name: string; description: string; spec: AgentSpec; createdAt: string; updatedAt: string }`. Lives in `packages/shared/src/agent.ts`, re-exported from the barrel.
- **Out of scope (do NOT implement):** agent deletion, API key encryption/persistence, dropping `agent_spec_snapshot` columns, chat search/filter, agent duplication, multi-user/auth, URL routing.
- **IDs:** `nanoid()` for all new primary keys.
- **Commit discipline:** One commit per task minimum, at the task's final step. Use conventional commit prefixes (`feat:`, `test:`, `refactor:`).

```typescript
    insert into agents (id, name, description, spec)

      coalesce(agent_spec -> 'identity' ->> 'description', 'Default agent'),

      await db.query(
        `insert into agents (id, name, description, spec) values ($1, $2, $3, $4)`,
```

```typescript
async function requestJson<T>(path: string, options?: RequestInit): Promise<T> {
  const response = await fetch(`${BASE_URL}${path}`, {
    ...options,
    headers: {
      "content-type": "application/json",
      ...(options?.headers as Record<string, string> | undefined)
    }
```

```tsx
                  onChange={(e) => setApiKey(e.target.value)}
                  placeholder="sk-..."
                />
              </label>
              <textarea
                rows={5}
                aria-label="Message"
                placeholder="Type your message..."
                value={message}
                onChange={(e) => setMessage(e.target.value)}
                onKeyDown={(e) => {
                  if (e.key === "Enter" && (e.metaKey || e.ctrlKey)) {
                    e.preventDefault();
                    handleSendMessage();
                  }
                }}
              />
              <button
                className="send-btn"
                onClick={handleSendMessage}
                disabled={sendState === "sending"}
              >
                {sendState === "sending" ? "Sending..." : "Send"}
              </button>
            </div>

            {error && <div className="error-banner">{error}</div>}

            {activeSession.latestTask && activeSession.taskMessages.length > 0 && (
              <details className="trace">
                <summary>Task Timeline</summary>
                {activeSession.taskMessages.map((tm) => (
                  <div key={tm.id} className="trace-item">
                    <span className={`trace-type trace-${tm.type}`}>{tm.type}</span>
                    <span className="trace-content">
                      {tm.content.slice(0, 200)}
                    </span>
                  </div>
                ))}
              </details>
            )}
          </div>
        )}
      </section>
    </main>
  );
}
```
