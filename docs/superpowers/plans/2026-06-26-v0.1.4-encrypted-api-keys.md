# Encrypted API Key Storage Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Store agent-scoped LLM API keys encrypted at rest (AES-256-GCM), decrypt just-in-time before runner calls, and remove the per-message API key from the client.

**Architecture:** A new `encryption` module in `apps/api` encrypts on write (agent create/update) and decrypts just-in-time inside `POST /api/chat-sessions/:id/messages`. The `agents` table gains a nullable `encrypted_api_key` column; `PgChatStore` exposes the encrypted value on agent rows and computes `hasApiKey` for responses. The web app moves the API key input from the chat panel to the create/edit agent forms.

**Tech Stack:** TypeScript, pnpm workspaces, Express + pg (api), React + Vite (web), zod (shared), vitest + pg-mem + supertest + @testing-library/react.

## Global Constraints

- API keys are encrypted at rest using AES-256-GCM with master key from `LLM_API_KEY_ENCRYPTION_KEY`.
- `LLM_API_KEY_ENCRYPTION_KEY` must be a 64-character hex string (32 bytes); validate with `/^[0-9a-f]{64}$/i`.
- Server refuses to start (throws) if `LLM_API_KEY_ENCRYPTION_KEY` is missing or invalid.
- Encrypted value format: `<iv>:<authTag>:<ciphertext>`, all base64 (iv = 16 random bytes, authTag = 16 bytes).
- Never log keys (plaintext, encrypted, or masked). Allowed logs are agent-id + event only.
- API responses never include key material — only computed `hasApiKey: boolean`.
- Decrypt only just-in-time, immediately before `runnerClient.runAgentTask()`.
- On decryption failure: respond 500 with exactly `"Failed to decrypt API key for agent"`; log full error server-side (no key data).
- Agents with `encrypted_api_key IS NULL`: respond 400 with exactly `"Agent API key not configured. Please update the agent settings."`
- `apiKey` is required (non-empty string) on `POST /api/agents`; reject empty with `"API key is required"`.
- `apiKey` is optional on `PUT /api/agents/:id`; when omitted/empty, leave `encrypted_api_key` unchanged.
- Preserve existing redaction behavior (`redactSecrets`, `redactUnknownJson`) for messages/logs.
- Use lowercase SQL and existing migration helpers (`addColumnIfNeeded`) to match the codebase.
- Run single-workspace tests with `pnpm --filter @agent-builder/<pkg> test`. `<pkg>` is `api`, `web`, or `shared`.

```typescript
// apps/api/src/encryption.ts
import crypto from "crypto";

const ALGORITHM = "aes-256-gcm";
const IV_LENGTH = 16;

function getMasterKey(): Buffer {
  const key = process.env.LLM_API_KEY_ENCRYPTION_KEY?.trim();
  if (!key) {
    throw new Error("LLM_API_KEY_ENCRYPTION_KEY environment variable is required");
  }
  if (key.length !== 64 || !/^[0-9a-f]{64}$/i.test(key)) {
    throw new Error("LLM_API_KEY_ENCRYPTION_KEY must be a 64-character hex string (32 bytes)");
  }
  return Buffer.from(key, "hex");
}

export function encryptApiKey(plaintext: string): string {
  const masterKey = getMasterKey();
  const iv = crypto.randomBytes(IV_LENGTH);
  const cipher = crypto.createCipheriv(ALGORITHM, masterKey, iv);
  const encrypted = Buffer.concat([cipher.update(plaintext, "utf8"), cipher.final()]);
  const authTag = cipher.getAuthTag();
  return `${iv.toString("base64")}:${authTag.toString("base64")}:${encrypted.toString("base64")}`;
}

export function decryptApiKey(encrypted: string): string {
  const masterKey = getMasterKey();
  const parts = encrypted.split(":");
  if (parts.length !== 3) {
    throw new Error("Invalid encrypted API key format");
  }
  const iv = Buffer.from(parts[0], "base64");
  const authTag = Buffer.from(parts[1], "base64");
  const ciphertext = Buffer.from(parts[2], "base64");
  const decipher = crypto.createDecipheriv(ALGORITHM, masterKey, iv);
  decipher.setAuthTag(authTag);
  return Buffer.concat([decipher.update(ciphertext), decipher.final()]).toString("utf8");
}

export function validateEncryptionKey(): void {
  // Throws if LLM_API_KEY_ENCRYPTION_KEY is missing or invalid. Used at server startup.
  getMasterKey();
}
```
